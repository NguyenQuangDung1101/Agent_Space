### 2. Main execution flow

```text
Receive user task
    ↓
Manager creates session
    ├── Generate session ID
    ├── Save original request
    └── Set status to RUNNING
    ↓
Manager loads catalogs
    ├── Registered execution agents from agent.json
    └── Registered shared tools from tool.json and config.json
    ↓
Call Analyze Agent
    ├── Identify objective and constraints
    ├── Determine required capabilities
    ├── Detect missing information
    └── Select execution solution
        ├── Single-agent solution
        └── Team solution
    ↓
Manager validates Analyze Agent output
```

#### Single-agent solution

```text
Analyze Agent selects:
    ├── One registered execution agent with tool (from registered tool list, not builtin tool of the agent) if needed
    ├── Agent task
    ├── Runtime system prompt when required
    └── Zero or more registered shared tools
    ↓
Manager validates:
    ├── Agent exists and is selectable
    ├── Tools exist
    ├── Agent call permission
    └── Runtime prompt requirements
    ↓
Agent Factory creates the selected runtime agent
    ↓
Tool Loader provides only assigned shared tools
    ↓
Execute selected agent
    ↓
Return its result directly
```

#### Team solution

```text
Manager starts the team execution pipeline
    ├── Planner Agent
    ├── Worker team
    └── Synthesizer Agent
    ↓
Planner receives:
    ├── Analyzed task
    ├── Registered execution-agent catalog
    ├── Registered shared-tool catalog
    └── Supported orchestration patterns
    ↓
Planner creates TeamPlan
    ├── Select registered worker agents
    ├── Define runtime roles and tasks
    ├── Assign zero or more shared tools to each worker (in registered tool list)
    ├── Define task dependencies
    └── Select orchestration:
        ├── Sequential
        ├── Parallel
        └── Supervisor
    ↓
Manager validates TeamPlan
    ├── Agent and tool IDs
    ├── Worker selectability
    ├── Task-member references
    ├── Dependencies and cycles
    ├── Runtime prompts
    └── Orchestration pattern
    ↓
Orchestrator creates isolated worker instances
    ↓
Orchestrator executes TeamPlan
    ├── Sequential:
    │   Run ready tasks one at a time in dependency order
    ├── Parallel:
    │   Run independent ready tasks concurrently
    └── Supervisor:
        Supervisor chooses the next validated worker and task
    ↓
Orchestrator manages:
    ├── Dependency outputs
    ├── Direct messages
    ├── Broadcast messages
    ├── Shared team context
    ├── Tool results
    └── Failures
    ↓
Synthesizer receives:
    ├── Original task
    ├── AnalysisResult
    ├── TeamPlan
    ├── Worker outputs
    └── Errors and tool results
    ↓
Synthesizer produces one final response
```

#### Persistence

```text
Execution result
    ↓
Save session files
    ├── Request
    ├── AnalysisResult
    ├── SingleAgentPlan or TeamPlan
    ├── Agent and tool assignments
    ├── Execution events
    └── Final response
    ↓
Persist graph relationships in Neo4j
    ├── System -[:REGISTERS]-> Agent
    ├── System -[:REGISTERS]-> Tool
    ├── System -[:RECEIVES]-> Task
    ├── Session -[:HAS_TASK]-> Task
    ├── Task -[:SOLVED_BY]-> Solution
    ├── SingleSolution -[:USES_AGENT]-> Agent
    ├── TeamSolution -[:USES_AGENT]-> Agent
    └── Solution -[:USES_TOOL]-> Tool
    ↓
Set session status to COMPLETED or FAILED
```
